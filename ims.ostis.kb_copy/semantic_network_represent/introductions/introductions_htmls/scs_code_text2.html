The SCs-code is linear version of the representation of the SC-code. The SCs-code is intended for the representation of sc-graphs(texts of the SC-code) in form of symbols sequence which can be edited with means of standard text editors or with means of the specialized sc.s-editor. Therefore, one of requirements, which can be produced towards the SCs-code, besides completeness and consistence, is a possibility to set up source texts without assistance of the specialized editor, using only symbols of standard keyboard.

In contrast to the SCn-code,which is also being the text version of sc-graphs mapping, formatting doesn’t matter in the SCs-code, all sentences could be written in one line.

In contrast to the SCg-code, there are not special notations for indications of structural type of SC-nodes, but there are images of sc-connectors, corresponding to a kernel and an extension of the SCg-code, as well as images of particular SC-connectors, for which there is no match in the SCg-code.

Technically, the SCs-code is the set of sc.s-texts. Each sc.s-text is a sequence of sc.s-sentences, each of which ends with the delimiter;; (double semicolon). Set of sc.s-sentences is divided into the set of simple sc.s-sentences andthe set of complex sc.s-sentences. Each complex sc.s-sentence contains built-in sentences in the composition. The built-in sentences are restricted by the limiter (*…*).

Comments are allowed within the sc.s- text of any level. It can be :

//one-line comment

/*multiline comment*/

At the beginning of the file, containing sc.s-text it is strongly recommended to specify the level and the version of the SCs-code. For this purpose, use the comment of a special kind:

/* SCs_code<"version">.Level<" level number "> */

For example:

/* SCs_code0.1.0.Level 6 */

It should be noted that the comment of the form /*...*/, used inside sc.s-frame (see the relevant section), isn’t omitted when parsing sc.s-text, and becomes part of the content of the corresponding sc-link. For example, when parsing a sentence

X =>r1: \[TEXT1/*COMMENT*/TEXT2\];;

Sc-link with contents TEXT1/*COMMENT*/TEXT2, ане TEXT1TEXT2 will be created.

The SCs-code is conditionally separated on 7 levels of difficulties. All the levels are equivalentto thepossibilitiesof knowledge representation, however texts of higher levels describe sc-graphs more laconically and convenient. Introduction of levels are meant to facilitate the work of the knowledge base developers at a set of sc.s-texts. It is recommended that the development ofthe knowledge bases includes using levels before the six inclusive.

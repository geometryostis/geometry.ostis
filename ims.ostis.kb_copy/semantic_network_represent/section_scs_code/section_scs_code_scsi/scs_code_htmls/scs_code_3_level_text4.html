It is possible to abbreviated record clarifying the types of sc-connectors for which we introduced sc.s-delimiters : and :: (modifiers sc-connectors) in sc.s-3 level sentences.

If these delimiters are found in a sentence several times, the record points that all the updates type refer to the same sc-connector.

You can use refinement type of sc-connectors, as part of non-atomic identifiers entered in the proposals, for example (v1 => r1 : r2: v2).

Using non-atomic identifiers such as <...>, {...}, \[* ... *\], \[...\] is not permitted.
